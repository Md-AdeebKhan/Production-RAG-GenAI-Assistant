DeepSeek RAG PDF Assistant

This project implements a Retrieval-Augmented Generation (RAG) system that enables users to upload PDF documents and query their contents through a conversational interface. The application retrieves relevant sections from the document using semantic search and provides context-aware responses generated by a language model.

The system combines document retrieval with large language models to improve response accuracy and ensure answers are grounded in the uploaded document.

Key Features
Upload and process PDF documents
Semantic document retrieval using vector embeddings
Context-aware question answering
Interactive user interface built with Streamlit
Integration with DeepSeek LLM for response generation
Technology Stack
Python
Streamlit
LangChain
Chroma Vector Database
HuggingFace Embeddings
DeepSeek API
System Workflow
The uploaded PDF is loaded and parsed.
The document is divided into smaller text chunks.
Each chunk is converted into vector embeddings.
Embeddings are stored in a Chroma vector database.
For each query, relevant chunks are retrieved.
The retrieved context is passed to the language model to generate the final response.
Installation

Install the required dependencies:

pip install -r requirements.txt

Create a .env file and add your API key:

DEEPSEEK_API_KEY=your_api_key

Run the application:

streamlit run app.py
Deployment

The application can be deployed using Streamlit Cloud by connecting the repository and configuring the API key in the deployment secrets.
